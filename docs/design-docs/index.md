# 设计文档索引

## 这个文件是做什么的

用于汇总所有设计类文档，告诉团队和 AI 应该去哪里找到关键设计决策。

## 适合写什么

- 各设计文档的目录和链接
- 每篇文档的主题说明
- 哪些文档是当前有效版本
- 阅读顺序建议

## 推荐用法

- 把它当成设计知识库的首页
- 新增设计文档后，先更新这里再通知团队
- AI 在做较大改动前，先从这里找到相关设计依据

## 设计文档列表

- [核心信念](core-beliefs.md)：团队坚持或禁止的技术原则
- [Init 模板规范](init-templates.md)：定义 `harness init` 命令需要补齐的所有文件模板内容，确保不同项目初始化后具有一致的文档格式和结构标准。
