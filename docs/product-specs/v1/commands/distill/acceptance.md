# `harness distill` 验收标准

- 成功路径：能够基于一个或多个文件、文件片段和用户说明生成 `.llms` 文件或目录内容，并返回统一 JSON 结果。
- 输入覆盖：必须支持文档、源码或二者混合输入；必须支持整文件和 `path:line-line` 形式的片段引用。
- 默认行为：默认调用宿主模型进行蒸馏，用户不需要传 `model` 参数。
- 交互行为：在初始输入不足以确定输出结构时，可通过交互方式继续收敛 `.llms` 内容与格式。
- 输出语义：产物必须体现面向下游 LLM / Agent 的结构化上下文包定位，而不是仅输出静态摘要；输出内容可包含摘要、关键关系、代码示例、文档精华和 Agent 使用指引。
- CLI 体验：前期命令面应保持极简，核心输入只要求文件、片段和说明；复杂选项不应成为主路径依赖。
- 失败路径：源文件不存在、文件无法读取、片段引用非法、宿主模型无法完成有效蒸馏、目标路径无法写入时失败。
- `--dry-run`：返回目标产物描述但不实际写入。
- Harness 边界：输入解析、输出结构化结果、目标路径和失败条件仍由 Harness 控制。
