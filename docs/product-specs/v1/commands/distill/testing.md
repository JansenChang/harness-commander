# `harness distill` 测试标准

- 必测：单文件输入、多文件混合输入、文件片段引用、文档与源码混合输入、带说明的宿主模型蒸馏、`--interactive` 对话收敛、`--dry-run`、`--json`。
- 必测：源文件不存在、文件不可读、片段范围非法、输出路径不可写等失败路径。
- 必测：默认情况下无需传 `model` 参数，且命令能按宿主模型默认路径执行。
- 必测：输出产物能够稳定体现 `.llms` 结构化上下文包语义，而非仅有静态摘要。
- 必测：输出中能够覆盖摘要、关键关系、参考单元中的至少一部分，并允许代码示例与文档精华共存。
- 通过标准：优先保证产物对下游 LLM / Agent 的可读性、可检索性和复用价值；其次保证 CLI 输入简洁、结果结构稳定、路径语义不漂移。
