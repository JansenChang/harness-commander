# `harness distill` 产品定义

## 产品定位

`harness distill` 用于从本地的文档、源码、文件片段以及用户补充说明中，借助宿主模型提取关键知识单元、分析跨文件关联，并生成供下游 LLM / Agent 使用的 `.llms` 结构化上下文包。

它不是单纯把长材料缩短，而是把原始材料重组为更适合模型检索、理解与复用的机器可读上下文。

## 解决的问题

- 原始文档、源码和历史材料会占用过多上下文，直接喂给模型成本高且噪声大。
- 多文件场景下，关键实现关系通常分散在多个类、模块或文档段落中，人工整理成本高。
- 下游 LLM / Agent 需要的不是静态文档，而是结构化、可检索、可复用的知识入口。

## 输入

- 一个或多个本地文件
- 文档与源码混合输入
- 整文件或指定行号片段
- 用户附加的一段说明，用于表达蒸馏目标

前期 CLI 只要求用户提供材料和意图，不要求用户在调用时显式指定复杂参数。

## 核心能力

- 根据用户提供的文件、片段和说明，默认调用宿主模型进行蒸馏。
- 分析跨文件的调用关系、类协作关系、实现模式和关键约束。
- 从源码中提取可复用代码示例，从文档中提炼高密度精华段落。
- 在信息不足时，通过多轮对话收敛输出内容、参数选型和 `.llms` 结构。
- 为后续 LLM / Agent 生成更高质量的上下文入口，而不是一次性 Prompt。

## 宿主模型角色

- `distill` 默认调用宿主模型。
- 用户不需要传 `model` 参数；模型选择属于运行环境，不属于业务参数。
- 宿主模型负责理解输入材料、识别关联关系、提取关键知识单元，并协助确定最终 `.llms` 内容。

## 输出

输出不局限于代码块，而是统一组织为可供下游模型消费的知识单元，典型包括：

- 高层摘要
- 关键关系与调用链说明
- 代码示例或文档精华片段
- 面向下游 Agent 的使用指引
- 推荐的输出结构或补充说明

最终产物可以是单个 `.llms` 文件，也可以是 `.llms` 目录中的结构化内容。

## 产物定位

`.llms` 的默认定位是结构化的通用上下文包，但长期目标是作为下游模型 / Agent 的知识检索索引与上下文入口。

典型使用方式包括：

- 作为 RAG 预处理后的高质量上下文资产
- 作为 Agent 调用某个工具或理解某个模块前的说明书
- 作为后续模型生成相似 API、模块或文档时的参考材料
